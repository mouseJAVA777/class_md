### TCP, HTTP, IPv4 Ethernet 프로토콜을 모두 이용한 패킷을 만들어 통신을 하려고 한다. 이 때 패킷을 만드는 과정(캡슐화)에 대해 설명하라

- ```
                                        HTTP + 데이터 
                            TCP + (HTTP + 데이터) 
                 IPv4 + (TCP + HTTP + 데이터)
  ```
  
  ethernet + (IPv4 + TCP + HTTP + 데이터 )
  
  데이터를 상위계층에서 하위 계층으로 전달할 때, 여러 프로토콜들로 캡슐화된 패킷을 전달하고, 다음 계층에서 새로운 헤더를 앞에 붙이는 식으로 진행됨.

### 2계층 Ethernet 프로토콜의 Ethernet Type의 역할을 간단히 설명하라

- Ethernet type에는 상위 계층의 프로토콜을 알려주는 바이트가 담겨져 있습니다. 예를 들어 IPv4의 경우 80 00 이 담겨져 있음

### 하나의 LAN 대역을 만들 때 주로 Star형 연결을 사용한다고 한다. 그 이유는?

- 비용적인 문제가 크다고 본다. mesh형의 경우가 더 안정적이지만, 하지만 모든 노드 간의 연결이 필요하므로 n개의 노드일 경우 n(n-1)/2 개의 노드 간의 케이블이 필요하다 하지만 Star의 경우 N개의 경우 가운데 중앙 장비 기준으로 N개의 케이블이 필요하므로 경제적이다.

### Mesh형 연결의 경우 어느 장비 하나가 고장난다 해도 전체적인 통신에는 문제가 없다고 한다. 그 이유는?

- 왜냐하면 다른 노드를 거쳐서 연결될 수 있기 때문이다. 예를 들어 a-b, b-c, a-c 로 연결되어 있다고 하자 만약 a-b가 끊어진다고 해도 a는 c를 거쳐 b와 연결될 수 있다.

### LAN과 WAN에 대해 설명하고, 차이점을 설명하시오

- LAN: 가까운 지역을 묶은 네트워크  
  WAN: 멀리 있는 지역을 묶어놓은 네트워크(LAN과 LAN을 묶음)

#### 2계층에서 사용하는 주소의 이름을 말하고 그 구성과 확인 방법을 서술하시오

- mac 주소, mac주소는 cmd창에서 ipconfig/all 이라는 명령어를 치고 물리적 주소 뒤에 16진수 12개 짜리 주소를 볼수 있습니다

### MAC address란?

- 장치의 물리적 주소로, 16진수 12개로 이루어져있다. 앞의 3바이트는 OUI, 뒤의 3바이트는 제조사에서 준 고유번호이다.

### 2계층 이더넷 헤더에는 어떤 정보들이 포함되어야 하는가?

- Destination Address 6바이트와 Source Address 6바이트, 그리고 Ethernet Type 2바이트로 이루어져있다. Destination Address는 목적지 주소, Source Address는 보내는 사람 주소, Ethernet Type은 상위 프로토콜의 미리보기이다.

### 네트워크, 인터넷, WWW 란?

- 네트워크 : 여러개의 시스템들이 데이터를 공유할 수 있게 하는 통신망

- 인터넷 : (International Network) 모든 컴퓨터를 하나의 통신망으로 연결(네트워크보다 작은 개념)

- WWW : 인터넷을 통해 웹 관련 데이터를 공유하는 것

### 패킷을 이용한 통신과정에서 상위계층에서 아래 계층으로 패킷을 만들어 통신하는 과정을 무엇이라고 하는가

- 캡슐호ㅏ

#### 2계층의 PDU와 3계층의 PDU는 각각 무엇인가요?

- 2계층: 프레임, 3계층: 패킷

#### TCP/IP와 OSI 7-Layer의 차이점을 얘기하시오

- TCP/IP 4계층은 네트워크인터페이스(물리, 데이터링크) 네트워크(네트워크) 전송(전송) 응용(세션, 표현, 응용) / 괄호가 7계층  
  으로 나누어져 있다. 실제 네트워크는 TCP/IP 4계층에 가까움

- TCP/IP - 4계층, 네트워크 통신을 기술에 따라 분류함  
  OSI - 7계층, 데이터의 흐름을 논리적으로 구간별로 분류함

### 2계층 프로토콜의 구성은?

- 수신 주소, 송신 주소, 이더넷타입, 페이로드

### 데이터에 TCP 만 붙어있는 형태의 4계층 PDU(Protocol Data Unit)를 부르는 또 다른 말은 무엇인가

- 세그먼트

### 2계층에서 하는 일은?

- LAN 대역에서 데이터 전송 + 오류제어 + 흐름제어

#### 네트워크 통신방식(유니캐스트, 멀티캐스트, 브로드캐스트) 에 대해 설명하시오

- 유니 캐스트는 1대1  
  멀티 캐스트는 1대N (특정 다수)  
  브로드 캐스트는 1대N (불특정 다수)

### 이더넷에 푸터가 뭐하는거임?

- FCS : 프레임에 문제가 있는지 판별에 사용

- CRC : 송신측에서 데이터로 부터 다항식에 의해 추출된 결과를 FCS에 담아 보내면 수신측에서 일치성 검사를 통해 오류 찾아냄
